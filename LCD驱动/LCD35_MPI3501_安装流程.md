# MPI3501 3.5inch RPi Display 安装流程

适用屏幕：

- MPI3501 3.5inch RPi Display
- 480x320
- XPT2046 / ADS7846 电阻触摸

## 为什么不能只跑原厂 LCD35-show

这次调试发现原厂 `LCD35-show` 在当前 Raspberry Pi OS / Debian 13 上有几个坑：

1. 新系统真正读取 `/boot/firmware/config.txt`，原厂脚本主要写 `/boot/config.txt`，直接运行后可能不生效。
2. 原厂脚本会尝试编译 `fbcp`，当前系统缺旧版 `bcm_host.h`，编译会失败；这个失败不影响 SPI LCD 本身工作。
3. 这块屏加载后是 Linux framebuffer：`/dev/fb0`，不是 HDMI/DSI 那种 DRM 显示器。
4. Wayland / WayVNC 依赖 DRM/KMS，不能直接拿这块 SPI framebuffer 当普通大屏使用。
5. 要让 LCD 像屏幕一样显示桌面，需要切到 X11 + fbdev/fbturbo，并让 VNC 使用 X11 service mode。

## 重装系统后的安装步骤

把本文件夹复制到树莓派桌面，保持目录结构：

```text
/home/intyu/Desktop/LCD驱动/
  LCD-show.tar.gz
  install_lcd35_mpi3501.sh
  LCD35_MPI3501_安装流程.md
```

执行：

```bash
cd /home/intyu/Desktop/LCD驱动
chmod +x install_lcd35_mpi3501.sh
./install_lcd35_mpi3501.sh
```

脚本会自动：

- 解压 `LCD-show.tar.gz`
- 安装 OpenCV、Picamera2、串口、X11 framebuffer/VNC 依赖
- 创建 `/home/intyu/env`
- 配置 `/boot/firmware/config.txt`
- 加载 `tft35a` LCD overlay
- 配置 XPT2046 / ADS7846 触摸校准
- 配置 X11 使用 `/dev/fb0`
- 配置 VNC 共享 LCD 桌面
- 禁用 WayVNC，启用 X11 VNC
- 重启

## 验证命令

```bash
cat /sys/class/graphics/fb0/name
cat /sys/class/graphics/fb0/virtual_size
cat /sys/class/graphics/fb0/bits_per_pixel
grep -A5 "ADS7846 Touchscreen" /proc/bus/input/devices
DISPLAY=:0 xrandr
```

正常结果应包含：

```text
fb_ili9486
480,320
16
ADS7846 Touchscreen
Screen 0 ... current 480 x 320
```

如果 LCD 仍显示终端：

```bash
sudo chvt 7
```

VNC 连接：

```text
10.81.42.136:5900
```

注意：VNC 共享的是 LCD 真实桌面，所以分辨率就是 `480x320`。

## 恢复大屏/HDMI

如果要恢复原来的大屏幕和 Wayland/WayVNC：

```bash
cd /home/intyu/Desktop/显示切换
./切换到大屏显示.sh
```

恢复 LCD：

```bash
cd /home/intyu/Desktop/显示切换
./切换到LCD显示.sh
```
